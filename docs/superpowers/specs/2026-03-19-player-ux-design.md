# Фаза 2: UX при просмотре — Design Spec

## Цель

Добавить OSD-оверлей с мини-списком каналов при переключении CH+/CH− и EPG-сайдбар с программой передач текущего канала, доступный по кнопке OK при видимом OSD.

## Контекст

Текущее состояние: при нажатии CH+/CH− канал переключается, показывается короткая нотификация `Lampa.Noty.show()` с названием. Нет информации о соседних каналах, текущей передаче или прогрессе. Нет доступа к EPG из плеера — только через долгое нажатие → карточка → «Программа».

## Компоненты

### 1. OSD мини-список каналов

**Внешний вид:** Полупрозрачная панель в левом нижнем углу поверх видео. Показывает до 5 каналов — 2 выше текущего, текущий (подсвечен), 2 ниже. Если каналов меньше 5, показываются все доступные. Список зацикливается: после последнего канала идёт первый (для выбора соседей).

**Содержимое каждого элемента:**
- Номер канала (позиция в текущем списке, 1-based, с учётом blacklist)
- Название канала
- У текущего канала: название текущей передачи (из `epg.getCurrent()`), если EPG доступен

**Визуальный стиль:**
- `z-index: 9999` — поверх элементов Lampa player UI
- Фон: `rgba(0,0,0,0.82)` с `backdrop-filter: blur(10px)`
- Скруглённые углы: `border-radius: 10px`
- Текущий канал: подложка `rgba(255,255,255,0.1)`, номер в красном бейдже (`#e63946`)
- Соседние каналы: `opacity: 0.4`
- Анимация появления: slide-in слева, `transform: translateX(-100%) → translateX(0)`, длительность `300ms`, `ease-out`
- Анимация исчезновения: fade-out, `opacity: 1 → 0`, длительность `300ms`, `ease-in`

**Поведение:**
- Появляется при CH+/CH− (keyCode 166/167, fallback PageUp/PageDown 33/34)
- Таймер 3 секунды, сбрасывается при каждом нажатии CH+/CH−
- Через 3 сек без действий — плавно исчезает (fade-out)
- Back/Escape — скрывает немедленно (без анимации)
- OK (Enter) пока OSD виден — открывает EPG-сайдбар

### 2. EPG-сайдбар

**Внешний вид:** Панель справа, ~40% ширины экрана. Видео продолжает играть слева.

**Содержимое:**
- Заголовок: название канала
- Список передач (из `epg.getAll(channelId)`):
  - Прошедшие: приглушённые (`opacity: 0.35`)
  - Текущая: подсвечена красным бордюром слева, красный текст времени с меткой «Сейчас», прогресс-бар
  - Будущие: нормальный текст

**Визуальный стиль:**
- `z-index: 10000` — поверх OSD мини-списка
- Фон: `rgba(0,0,0,0.88)` с `backdrop-filter: blur(12px)`
- Анимация появления: slide-in справа, `transform: translateX(100%) → translateX(0)`, длительность `300ms`, `ease-out`
- Анимация исчезновения: slide-out вправо, `transform: translateX(0) → translateX(100%)`, длительность `250ms`, `ease-in`
- Текущая передача: `border-left: 3px solid #e63946`, фон `rgba(230,57,70,0.2)`
- Прогресс-бар: фон `rgba(255,255,255,0.15)`, заполнение `#e63946`, высота `2px`

**Пустое состояние EPG:** Если `epg.getAll(channelId)` возвращает пустой массив, сайдбар показывает сообщение «Нет данных о программе» по центру вместо списка передач.

**Поведение:**
- Открывается по OK (Enter) при видимом OSD мини-списке
- ↑/↓ — скролл по списку передач (фокус перемещается по элементам)
- ← или Back — закрывает сайдбар
- CH+/CH− при открытом сайдбаре — переключает канал, сайдбар обновляется на программу нового канала
- Автоскролл к текущей передаче при открытии
- Длинные списки (>50 передач): виртуализация не требуется (EPG редко превышает 30-40 записей на день)

## Управление клавишами (приоритет слоёв)

**Важно: сосуществование с нативным плеером Lampa.** OSD-обработчик перехватывает клавиши на capture-фазе (`addEventListener(..., true)`) и вызывает `e.stopPropagation()` + `e.preventDefault()` **только для тех клавиш, которые он обрабатывает на текущем слое**. Все остальные клавиши проходят к нативному плееру без изменений. На базовом слое (ничего не показано) перехватываются только CH+/CH− — OK, →, ↑/↓ и все остальные клавиши остаются у нативного плеера (пауза, перемотка, громкость и т.д.).

**Слой 3 — EPG-сайдбар открыт (высший приоритет):**
| Клавиша | Действие |
|---------|----------|
| ↑/↓ | Скролл по списку передач |
| ← / Back | Закрыть сайдбар |
| CH+/CH− | Переключить канал, обновить сайдбар |
| OK, → и др. | Не перехватываются |

**Слой 2 — OSD мини-список виден:**
| Клавиша | Действие |
|---------|----------|
| CH+/CH− | Переключить канал, обновить OSD, сбросить таймер |
| OK (Enter) | Открыть EPG-сайдбар |
| Back | Скрыть OSD немедленно |
| →, ↑/↓ и др. | Не перехватываются (проходят к нативному плееру) |

**Слой 1 — ничего не показано (базовый):**
| Клавиша | Действие |
|---------|----------|
| CH+/CH− | Переключить канал, показать OSD |
| Все остальные | Не перехватываются — проходят к нативному плееру Lampa (OK=пауза, →=перемотка и т.д.) |

## Архитектура файлов

| Файл | Изменение | Описание |
|------|-----------|----------|
| `src/ui/osd.js` | Новый | OSD мини-список + EPG-сайдбар. Экспортирует `createOsd(channels, onSwitch)` |
| `src/style.js` | Модифицировать | Добавить CSS для `.liptv-osd-*` и `.liptv-epg-sidebar-*`, анимации |
| `src/index.js` | Модифицировать | Инициализировать OSD при старте плеера, передавать список каналов. Убрать текущий `Lampa.Noty.show()` при переключении. Рефакторинг keydown-обработчика: вынести обработку клавиш в `osd.js`, в `index.js` оставить только создание/уничтожение OSD и проброс списка каналов |

### API модуля `src/ui/osd.js`

```javascript
// Создаёт OSD и сайдбар, привязывает к DOM
// channels — текущий отфильтрованный список каналов
// onSwitch(channel) — callback переключения канала
createOsd(channels, onSwitch)

// Возвращаемый объект:
{
  show(currentIndex),      // показать OSD с текущим каналом
  hide(),                  // скрыть OSD немедленно
  showEpgSidebar(channel), // показать EPG-сайдбар
  hideEpgSidebar(),        // скрыть EPG-сайдбар
  destroy()                // очистить DOM, таймеры, слушатели
}
```

## Обработка клавиш — рефакторинг

Текущий keydown-обработчик в `index.js` (блок `// CH+/CH- channel switching during playback`) обрабатывает только CH+/CH−. С добавлением OSD и EPG-сайдбара логика становится многослойной (3 слоя приоритетов). Чтобы не раздувать `index.js`:

- **`osd.js` берёт на себя** клавиатурную обработку во время воспроизведения: CH+/CH−, OK, Back, ←, ↑/↓ (только на слоях 2-3, где OSD/сайдбар видны)
- **`osd.js` регистрирует** свой `document.addEventListener('keydown', ..., true)` при создании и снимает при `destroy()`
- **`index.js` удаляет** свой текущий keydown-обработчик CH+/CH− и делегирует переключение каналов через `onSwitch` callback
- **`index.js` отвечает** только за: создание OSD при начале воспроизведения, уничтожение при закрытии плеера, передачу **уже отфильтрованного** списка каналов (без blacklist) и callback'а
- **Blacklist-фильтрация** происходит в `index.js` при создании OSD и при переключении канала. `osd.js` не знает о blacklist и не импортирует `storage`

## Жизненный цикл

1. Пользователь начинает воспроизведение канала → `index.js` создаёт OSD через `createOsd(channels, onSwitch)`
2. OSD регистрирует keydown-обработчик
3. CH+/CH− → OSD вызывает `onSwitch(channel)`, показывает мини-список
4. Через 3 сек → OSD скрывается автоматически (fade-out)
5. OK при видимом OSD → OSD показывает EPG-сайдбар
6. Плеер закрывается (`Player.listener 'destroy'`) → `osd.destroy()` снимает обработчик

## Вне скоупа (Фаза 2)

- Ввод номера канала цифрами — не нужен (нет цифровых кнопок на пульте)
- EPG-грид (таблица каналы×время) — возможно в Фазе 3
- Использование `Lampa.Template` / `Lampa.Controller` / `Lampa.Network` — Фаза 1
- Мультиплейлист, catch-up — Фаза 3
